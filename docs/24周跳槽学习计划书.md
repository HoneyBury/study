# 24周跳槽学习计划书

## 1. 目标定位

当前目标不是从零转行，而是完成一次有针对性的职业升级：

- 从：`Qt/C++ 上位机开发`
- 升级为：`工业视觉 + 运动控制 + 边缘 AI 部署` 方向的系统型工程师

6个月后的目标岗位优先级：

- `工业视觉 C++ 工程师`
- `视觉定位与运动控制工程师`
- `边缘 AI / 推理部署工程师`
- `智能装备软件工程师`
- `机器人感知控制集成工程师`

## 2. 时间预算

已知可投入时间：

- 周一到周五每天约 `5` 小时
- 周六周日每天约 `8` 小时
- 每周总投入约 `41` 小时
- 24周总投入约 `984` 小时

建议每周固定分配：

- `18` 小时：主项目开发
- `10` 小时：系统学习新知识
- `5` 小时：算法部署实验与验证
- `4` 小时：简历/面试/岗位研究
- `4` 小时：复盘、文档、知识整理

## 3. 六个月总交付物

24周结束时，必须至少拿出以下成果：

- 一个主项目：`工业视觉引导定位与运动执行系统`
- 一个补充项目：`C++ ONNX Runtime/TensorRT 推理封装`
- 一套项目文档：架构图、模块说明、误差链路、性能指标、异常恢复设计
- 一份正式简历
- 两个定向版本简历：
  - 偏工业视觉/运动控制
  - 偏边缘 AI / 推理部署
- 一套面试讲解材料：
  - 项目背景
  - 架构设计
  - 难点拆解
  - 性能优化
  - 稳定性设计
- 一份岗位投递与反馈记录表

## 4. 项目定义

### 4.1 主项目

项目名称建议：

- `工业视觉引导定位与运动执行系统`

项目目标：

- 模拟或抽象真实工业设备场景
- 接入相机图像
- 完成标定、畸变矫正、目标定位、坐标转换
- 将视觉定位结果转换为运动执行指令
- 通过状态机组织完整流程
- 支持日志、配置、异常恢复、结果回放
- 具备可扩展的算法接口或推理模块

建议模块：

- 相机接入模块
- 标定与矫正模块
- 定位结果处理模块
- 坐标变换模块
- 运动控制抽象层
- 状态机调度模块
- 配置中心
- 日志与故障诊断模块
- 数据回放与可视化模块

### 4.2 补充项目

项目名称建议：

- `Vision Inference SDK`

项目目标：

- 使用 Python 侧完成模型验证
- 导出 ONNX
- 使用 C++ + ONNX Runtime 完成推理
- 支持图片/视频/相机输入
- 输出耗时、FPS、资源占用
- 形成可展示的推理封装能力

## 5. 执行原则

- 不追求同时学习多个热点方向
- 主线始终围绕：`视觉 + 控制 + 部署`
- 每周必须有代码交付，不接受纯看课
- 每两周至少产出一次可见成果
- 第16周前完成主项目第一个可演示版本
- 第17周开始准备求职材料
- 第19周开始小规模试投
- 第21周开始正式集中投递
